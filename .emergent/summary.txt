<original_problem_statement>
The user wants to design and create a mobile application to manage travel expenses for their company's employees.

**PRODUCT REQUIREMENTS:**
- **User Profiles & Roles:**
  - **Worker:** Can create travel projects, add companions, and log expenses. Can only edit/delete their own expenses.
  - **Authorizer:** Can approve created trips. Expenses cannot be logged until a trip is approved.
  - **Administrator:** Can view all information, export data to Excel, manage users, and edit any user's expenses.
- **Authentication:** The user initially requested Microsoft Outlook auth but accepted a custom JWT (email/password) solution after learning Microsoft auth was not available.
- **User Management:** The administrator is responsible for creating user accounts and assigning roles.
- **Trips/Projects:**
  - Each trip is a project.
  - Trips must be assigned to one of ~15 predefined cost centers.
  - A trip can have one or more employees. The creator adds companions, who then get access to the trip.
- **Expense Logging:**
  - Must be linked to an approved trip.
  - Use the device's camera to capture and attach images of tickets/receipts.
  - Record the following for each expense: amount, date, establishment, and reason for the expense (selected from a predefined, editable list).
- **Notifications:** The app should send notifications when a trip's status changes (e.g., approved).
- **Data Export:** The administrator must be able to export a report in Excel format containing: User, Cost Center, Date, Expense Amount, and Expense Reason.
- **Audit Trail:** A history of changes should be recorded.
</original_problem_statement>
